# Reward Token Security (Smart Contract)

Security-focused reward distribution contract.

## Features

- prevents double reward claims
- verifies battle winner before reward claim
- max reward limit per battle
- total reward pool cap
- secure state tracking
- Hardhat test cases

## Security Concepts Practiced

- access control validation
- replay attack prevention
- reward abuse prevention
- safe token transfer validation

## Stack

Solidity  
Hardhat  
Ethers.js  
Mocha / Chai  

## Purpose

This project demonstrates how to design secure reward distribution logic for Web3 gaming or DeFi applications.

Note: This project originated from a smart contract security assessment and was extended with additional validation checks and tests for portfolio purposes.
